# Sotto

Локальная push-to-talk диктовка для macOS (Apple Silicon). Зажимаешь **правую
Option** в любом приложении, говоришь, отпускаешь — распознанный текст
вставляется в активное поле ввода. Распознавание полностью локальное и офлайн
(mlx-whisper); опционально — LLM-пост-обработка через OpenAI-совместимый API.

*Sotto — от итал. sotto voce, «вполголоса».*

## Установка

### Вариант 1: готовый Sotto.app (рекомендуется)

Скачайте `Sotto-<версия>-arm64.dmg` со страницы
[Releases](https://github.com/thothlab/sotto/releases), откройте образ и
перетащите `Sotto.app` в `Applications`. Сборка не нотаризована Apple,
поэтому перед первым запуском снимите карантин:

```bash
xattr -dr com.apple.quarantine /Applications/Sotto.app
```

(либо правый клик по Sotto.app → Open → Open). Права macOS в этом случае
выдаются самому Sotto.app, Python и терминал не нужны.

### Вариант 2: из исходников

```bash
./install.sh
```

Скрипт создаёт `.venv`, ставит зависимости и проверяет права macOS.

В обоих вариантах модель (`large-v3-turbo`, ~1.6 ГБ) скачивается при первом
запуске в `~/.cache/huggingface/`, дальше — полный офлайн.

Требования: Apple Silicon, macOS 14+; для варианта 2 — Python 3.11+.
`ffmpeg` не обязателен: Sotto декодирует свои записи сам; поставить
`brew install ffmpeg` стоит для прогона тестов и на будущее — для
распознавания готовых аудиофайлов не-WAV форматов.

## Права macOS

При запуске готового приложения права выдаются **Sotto.app**; при запуске из
исходников — **терминалу-родителю** (Terminal / iTerm2). В System Settings →
Privacy & Security включите:

| Право | Зачем | Без него |
|---|---|---|
| **Microphone** | запись голоса | запись не стартует (запрос появится сам при первой записи) |
| **Input Monitoring** | перехват клавиши-триггера | зажатие правой Option не замечается |
| **Accessibility** | эмуляция Cmd+V | текст только копируется в буфер, вставлять придётся вручную |

Проверка статуса: `./.venv/bin/python -m sotto check`.
После изменения галочек **перезапустите терминал**.

## Запуск

```bash
./run.sh
```

В menu bar появится 🎙. Зажмите правую Option — иконка станет 🔴 (с
индикатором уровня звука), говорите, отпустите — ⏳, и через ~1–2 секунды
текст вставится в активное поле.

Меню: статус · пауза · выбор модели · выбор микрофона · переключатель
пост-обработки · последние 5 транскриптов (клик — скопировать) · выход.

### Распознавание готового файла

```bash
./.venv/bin/python -m sotto transcribe запись.wav     # 16 kHz mono WAV — напрямую
./.venv/bin/python -m sotto transcribe голос.m4a      # прочие форматы — через ffmpeg
# из standalone-приложения:
/Applications/Sotto.app/Contents/MacOS/Sotto transcribe голос.m4a
```

Для не-WAV форматов нужен `ffmpeg` (`brew install ffmpeg`) — без него Sotto
подскажет, что поставить.

### Автозапуск

```bash
./.venv/bin/python -m sotto install-autostart    # LaunchAgent com.sotto.app
./.venv/bin/python -m sotto uninstall-autostart
```

При автозапуске права macOS должны быть выданы самому бинарю Python из
`.venv` (macOS покажет запросы при первом срабатывании).

## Настройка

Конфиг создаётся при первом запуске: `~/.config/sotto/config.toml`.

```toml
trigger_key = "alt_r"          # или "f19", "cmd+shift+d", одиночный символ
vocabulary = ["Claude Code", "MCP"]   # термины: подсказка Whisper + словарь LLM

[model]
name = "mlx-community/whisper-large-v3-turbo"
language = ""                  # "" = авто (русский/английский вперемешку — ок)

[stt]                          # где распознавать речь
backend = "local"              # "local" (офлайн, mlx-whisper) | "cloud"
cloud_base_url = "https://api.openai.com/v1"
cloud_api_key = ""             # или env OPENAI_API_KEY / GROQ_API_KEY
cloud_model = "whisper-1"

[postprocess]                  # выключено по умолчанию
enabled = false
base_url = "https://openrouter.ai/api/v1"
api_key = ""                   # или env SOTTO_API_KEY / OPENROUTER_API_KEY
model = "openai/gpt-4o-mini"
```

Пост-обработка: сырой транскрипт прогоняется через LLM (пунктуация,
капитализация, доменные термины из `vocabulary`). При недоступности API молча
вставляется сырой текст. Ключ — только из конфига или env, в коде его нет.

### Выбор модели: локально или облако

Локальные модели переключаются в меню «Модель» (размер = сколько скачается
при первом использовании): `large-v3-turbo` 1.6 ГБ (дефолт, лучшее качество
RU/EN) · `medium` 1.5 ГБ · `small` 0.5 ГБ · `tiny` 0.15 ГБ.

Не хочется качать гигабайты — переключитесь на **облачное распознавание**
(меню «Модель» → «Облако» или `backend = "cloud"` в конфиге): аудио уходит на
OpenAI-совместимый endpoint `/audio/transcriptions`.

| Провайдер | cloud_base_url | cloud_model |
|---|---|---|
| OpenAI | `https://api.openai.com/v1` | `whisper-1` или `gpt-4o-mini-transcribe` |
| Groq (быстрый, есть free tier) | `https://api.groq.com/openai/v1` | `whisper-large-v3-turbo` |

**OpenRouter распознавание речи не предоставляет** — его ключ подходит только
для секции `[postprocess]` (пунктуация/словарь). Учтите: при облачном бэкенде
диктовки покидают ваш Mac; офлайн-режим — только `local`.

## Сборка релиза

```bash
make app     # локально: dist/Sotto.app + dist/Sotto-<версия>-arm64.dmg
```

Публикация на GitHub: поднять версию в `sotto/__init__.py`, затем

```bash
git tag v0.2.0 && git push origin v0.2.0
```

GitHub Actions (`.github/workflows/release.yml`, arm64-runner) прогонит линт
и тесты, соберёт Sotto.app и приложит zip к автоматически созданному Release.

## Troubleshooting

- **Выдал права, но ничего не изменилось.** Права macOS применяются только к
  новому процессу: после любых галочек в Privacy & Security **перезапустите
  Sotto** (и терминал, если запускаете из исходников).
- **Зажимаю Option — ничего не происходит.** Нет Input Monitoring.
  `sotto check`, поставьте галочку, перезапустите.
- **Текст не вставился.** Нет Accessibility (macOS дропает эмуляцию Cmd+V
  молча) — Sotto оставит текст в буфере и покажет уведомление, а в иконке
  появится ⚠️; вставьте Cmd+V вручную, выдайте право, перезапустите Sotto.
  Текст также доступен в меню «Последние транскрипты».
- **Иконка долго показывает ⏳, меню не открывается.** Идёт первая загрузка
  модели в память (после запуска или смены модели) — приложение может не
  отвечать на клики десяток-другой секунд. Дождитесь 🎙 — статус в меню
  покажет «Готов».
- **«Микрофон недоступен» в статусе.** Право Microphone не выдано или занято
  устройство. Проверьте выбор микрофона в меню.
- **Первая фраза распознаётся долго.** Идёт скачивание/прогрев модели — статус
  в меню «Загрузка модели…». Дальше ~1–2 сек.
- **Распознаёт с ошибками доменные термины.** Добавьте их в `vocabulary` —
  они подсказываются Whisper даже без пост-обработки.
- **Логи автозапуска:** `/tmp/sotto.out.log`, `/tmp/sotto.err.log`.

## Design decisions

- **Python + mlx-whisper, не Swift.** mlx-whisper — Python-native, оптимизирован
  под Apple Silicon; pynput даёт надёжный глобальный перехват клавиш через
  Quartz event tap; rumps закрывает menu bar. Swift потребовал бы в разы больше
  кода и ручных биндингов whisper.cpp при том же результате.
- **`large-v3-turbo`, не `medium`.** На M-серии turbo транскрибирует 10–15 с
  речи за ~1–1.5 с — укладывается в цель ≤2 с, а на смешанном RU/EN качество
  заметно выше medium. Модель держится в памяти между вызовами + прогревается
  при старте, поэтому загрузка весов не попадает в латентность.
- **События клавиатуры не подавляются.** Правая Option сама по себе ничего не
  печатает, поэтому подавление не нужно; левая Option и раскладки не
  затрагиваются. Если назначить символьную клавишу триггером, её нажатие
  дойдёт и до активного приложения — ограничение v1.
- **Floating-оверлей не делался** (в ТЗ опционален): уровень звука показывается
  прямо в menu bar (🔴▅) — без второго GUI-слоя.
- **Восстановление буфера — только текст** (`NSPasteboardTypeString`).
  Картинки/файлы в буфере после вставки не восстанавливаются — ограничение v1.
- **Пока идёт распознавание, новая запись не стартует** — упрощает порядок
  вставки; на практике транскрипция быстрее, чем пауза между фразами.
- **Streaming-распознавания нет** (как и в референсе): текст появляется после
  отпускания клавиши.
- **Содержимое диктовок не логируется**; WAV пишется во временный файл и
  удаляется сразу после распознавания.
- **Standalone-сборка — PyInstaller, без torch.** mlx-whisper объявляет torch
  зависимостью, но в рантайм-пути транскрипции он не импортируется (нужен
  только конвертеру PyTorch-чекпойнтов) — исключён из бандла (−437 МБ).
  WAV декодируется собственным кодом (wave + numpy), поэтому системный ffmpeg
  приложению не нужен. Сборка подписана ad-hoc, не нотаризована — при
  установке из релиза нужно снять карантин (см. «Установка»).
